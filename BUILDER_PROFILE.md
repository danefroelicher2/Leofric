# Leofric — Builder Profile and Project Purpose

## Who Is Building This

- CS degree, minimal professional Python experience, transitioning actively into Python mastery
- Two years of JavaScript experience at Kroger (professional, production environment)
- Currently drilling Python daily with a structured program targeting technical interview readiness
- Not a beginner in logic or engineering thinking — a beginner specifically in Python depth and systems-level work
- Two prior iOS apps shipped — familiar with App Store Connect, Apple developer ecosystem, not starting from zero on mobile
- Based in the US, building on personal hardware at home

## Why This Project Exists

Leofric is not a portfolio checkbox. It is a working system that lives in the builder's home, gets used daily, and grows in capability over time. The project exists for three reasons simultaneously:

**One — career positioning.** The target employers are defense contractors, robotics companies, and intelligence-adjacent organizations — Palantir, Anduril, Leidos, and similar. These employers have seen web apps and todo lists. They have not seen a junior developer who built a distributed home intelligence system from scratch, deployed it on physical hardware, and can explain every architectural decision under pressure. Leofric is the proof of work that bridges the gap between a non-target school, a modest GPA, and the room these companies operate in.

**Two — genuine daily use.** A project built and abandoned is a project that cannot be spoken about with depth. Leofric runs at the front door, watches the house, and alerts the builder's phone. It is used every day. That daily use produces the kind of genuine familiarity that surfaces in interviews and cannot be faked.

**Three — Python growth.** Every module of this system requires real Python — data structures, concurrency, APIs, pipelines, ML inference. Building Leofric is not separate from Python mastery. It is the applied layer on top of it. The goal is to be the person in the room who can think when the tools are taken away.

## The Long-Term Target

The immediate goal is to join a defense, robotics, or intelligence-adjacent company as a technical contributor. The long-term goal is a role at the CIA or similar intelligence organization through demonstrated technical expertise. The path runs through real work, not credentials. Leofric is the first serious step on that path.

## Career Context for Claude Code

When making architectural or implementation decisions, always consider whether the choice demonstrates serious engineering judgment to a defense or robotics recruiter. Prefer solutions that show understanding of real systems — sensor fusion, edge compute, distributed coordination, anomaly detection — over solutions that are merely functional. The bar is not "does it work." The bar is "would an Anduril engineer find this interesting."

## What Success Looks Like

- Leofric is running in the home, used daily, genuinely useful
- The builder can open a laptop in an interview, show the system live, and explain every decision
- The GitHub repo tells a coherent story of an engineer who thinks seriously about hard problems
- The iOS app is on the builder's phone and demonstrable in thirty seconds
- When a recruiter at a defense company asks what the builder has built, the answer takes five minutes to explain properly and leaves the recruiter wanting to know more
